Informations supplémentaires sur Teo Linux/X11

Installation

La configuration nécessaire pour faire tourner l'émulateur est la configuration standard des distributions récentes; seule la présence obligatoire de la version 2.12 ou supérieure du toolkit GTK+ peut nécessiter une mise à jour du système (RedHat 5.x, SuSE 5.x), disponible ici.

Vous pouvez installer l'émulateur de deux façons:

  • Exécuter teo-1.8.1-i586.deb. Le lancement de Teo se fera par le menu Application/Jeux/Emulateur Teo.
  • Extraire teo-1.8.1-i586.tar.gz. Le programme éxécutable se nomme teo.
Lancement

Teo accepte les paramètres supplémentaires suivants sur la ligne de commande:

[-help]affiche l'aide en ligne
[-geometry +x+y]spécifie la position de la fenêtre
[-noshm]désactive l'utilisation de l'extension MIT-SHM
[-display displayname]spécifie le nom du serveur X à utiliser
[xxxxx]charge un (des) fichier(s) SAP, K7, M7

L'installation par les paquets .deb lance Teo avec l'option -loadstate par défaut. Pour désactiver cette option ou en ajouter d'autres, choisir Système/Préférences/Menu principal, sélectionner alors Jeux/Emulateur Teo, cliquer sur Propriétés et changer les options par défaut de l'exécutable.

Clavier

Si la touche ALTGR reste sans effet, vérifez vos réglages pour l'agencement du clavier en ce qui concerne les touches de 3ème niveau: Système/Préférences/Clavier/Agencement/Options de l'agencement.../Touche sélectionnant le 3ème niveau.

Générateur sonore

Teo utilise le périphérique sonore par défaut du système (via ALSA).

Activer l'accès direct aux disquettes sur les systèmes Linux récents

Sur les systèmes récents, les lecteurs de disquette ne sont pas actifs à l'installation. Il est nécessaire de procéder à quelques ajustements à partir d'un Terminal :

Créer l'entrée pour le premier lecteur (si vous en avez un) :

  • Exécuter sudo mkdir /media/floppy0
  • Exécuter sudo gedit /etc/fstab
  • Ajouter la ligne /dev/fd0 /media/floppy0 auto rw,user,noauto,exec 0 0 si aucune ligne ne commence par /dev/fd0
  • Enregistrer le fichier et quitter GEdit

Créer l'entrée pour le deuxième lecteur (si vous en avez un) :

  • Exécuter sudo mkdir /media/floppy1
  • Exécuter sudo gedit /etc/fstab
  • Ajouter la ligne /dev/fd1 /media/floppy1 auto rw,user,noauto,exec 0 0 si aucune ligne ne commence par /dev/fd1
  • Enregistrer le fichier et quitter GEdit

Déclarer les accès disquette :

  • Exécuter sudo gedit /etc/modules
  • Ajouter la ligne floppy si elle n'existe pas
  • Enregistrer le fichier et quitter GEdit
  • Exécuter sudo adduser $USER floppy

Et pour finir, activer Le tout :

  • Redémarrer l'ordinateur

Si toutefois la lecture directe a des difficultés à fonctionner (erreurs d'entrée/sortie, par exemple), cliquez avec le bouton droit sur Lecteur de disquettes dans votre navigateur de fichiers et sélectionnez Détecter le média.

Désinstallation

Si l'installation a été faite avec les fichiers tar.gz, il suffit d'éliminer le répertoire teo racine.

Si l'installation a été faite avec les fichiers .deb, exécuter sudo apt-get remove --purge teo dans un Terminal. Il se peut que l'entrée dans le menu Applications/Jeux/Emulateur Teo ne disparaisse pas. Exécuter alors rm ~/.local/share/applications/teo.desktop. Et pour vraiment tout nettoyer, exécuter rm -r ~/.teo, qui est le répertoire contenant les fichiers teo.cfg et autosave.img.
